# Basic Concepts

This file explains the most fundamental concepts used in the Tradecraft application.

## Trading
- **Order**: Instruction to buy or sell an asset.
- **Position**: Current holdings in an asset.
- **Portfolio**: Collection of positions.
- **Market Data**: Information about prices, volume, etc.

## Application Structure
- **Module**: A folder or file containing related code.
- **API**: Interface for communication between frontend and backend.
- **Broker**: Service that executes trades.
- **Strategy**: Logic that decides when to buy/sell.
- **Risk Management**: Logic to protect against losses.
- **Alert**: Notification sent to users.

## Data Flow
- Data is fetched, processed, and used for trading decisions.
- ML models use features extracted from data to make predictions.

## Testing
- Ensures code works as expected.
